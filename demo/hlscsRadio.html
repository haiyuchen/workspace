合同基本信息示例

效果

应用1：
承租人以□【现金】□【银行转账】方式支付租金
应用2：
承租人以方式支付租金
应用3：
出租人证件类型：□【居民身份证】□【社会统一信用代码】□【】

开发代码

应用1： <div>承租人以<a name="payType" data-radio="02" target-type="radio" fill-style="diamond">□</a>【现金】<a name="payType" data-radio="01">□</a>【银行转账】方式支付租金</div> 应用2： <div>承租人以<input name="payType" data-replace="payment" target-type="radio">方式支付租金</div> 应用3： <div>出租人证件类型：<span name="entJcardType" data-radio="01" target-type="radio" fill-style="diamond">□</span>【居民身份证】<span name="entJcardType" data-radio="08">□</span>【社会统一信用代码】<span name="entJcardType" data-radio="other">□</span>【<input name="basicInput1" />】</div>

开发说明

应用1：仅需要选项标签
多个选项，使用a或span标签，name属性值添加同一个合同基本信息中的对应字段，标签内里面填入需要的非选中样式，标签属性添加data-radio属性，值为选项对应的选项值。
组件的首个a或span标签添加target-type="radio"标识，添加fill-style属性，值为选定的样式值，如下：
tick: { 非选中样式: ' ', 选中样式: '✔' }
diamond: { 非选中样式: '□', 选中样式: '■' }
应用2：仅需要转换后的文本
使用input标签，name属性值添加合同基本信息中的对应字段，添加target-type="radio"标识，添加data-replace属性，值为使用的约定数据，约定数据请参考“合同数据.xlsx”使用。
应用3：需要有“其他”选项，并且其他选项的文本可编辑
同理应用1，“其他”选项的data-radio值为“other”，而编辑的文本使用basicInput组件
“其他”选项将涵盖现有选项（01、08）以外的所有值（02、03、04、。。。、other）


扩展信息示例

应用1：
承租人以□【现金】□【银行转账】方式支付租金

承租人以□【现金】□【银行转账】方式支付租金□【】

开发代码

应用1： <div>承租人以<a name="hlscsRadio1" data-radio="01" target-type="radio" fill-style="diamond">□</a>【现金】<a name="hlscsRadio1" data-radio="02">□</a>【银行转账】方式支付租金</div> 应用2： <div>承租人以<a name="hlscsRadio2" data-radio="01" target-type="radio" fill-style="diamond">□</a>【现金】<a name="hlscsRadio2" data-radio="02">□</a>【银行转账】方式支付租金<a name="hlscsRadio2" data-radio="other">□</a>【<input name="hlscsRadio2" radio-input="otherValue" />】</div>

开发说明

应用1：仅需要选项标签
多个选项，使用a或span标签，name属性值添加同一个不包含在合同基本信息中的随意字段，标签内里面填入需要的非选中样式，标签属性添加data-radio属性，值为选项对应的选项值。
组件的首个a或span标签添加target-type="radio"标识，添加fill-style属性，值为选定的样式值，如下：
tick: { 非选中样式: ' ', 选中样式: '✔' }
diamond: { 非选中样式: '□', 选中样式: '■' }
应用2：需要有“其他”选项，以及“其他”对应的填写文本框
同理应用1
“其他”选项的data-radio值为“other”
“其他”的编辑文本使用input标签，name为该组件的同一个字段，且需添加radio-input="otherValue"属性
